# 人脸实时框选 + 平滑跟随 + 身份标记 — 实现方案

> 目标：直播画面中，**每张人脸都用框圈住**，人脸移动时**框平滑跟随**，
> 框上方标注**「是谁」**（会员姓名 / 陌生人）。
> 约束：不加重后端推理负担（卡顿是既有痛点），框要"丝滑"不能一顿一顿。

---

## 0. 现状（为什么现在做不到）

| 环节 | 现状 | 问题 |
| --- | --- | --- |
| 后端 `face.py:410` | `rect = face_rects[0]` 只取第一张脸 | 多人只框一个 |
| 后端 `face.py:710` `_push_result` | 消息仅 `{type, member_id, name}` | **不含坐标**，前端无从画框 |
| 后端检测频率 | 调度每 5 帧 × 检测器每 3 帧 ≈ **每 15 帧一次**（约 1~2 次/秒） | 直接按此频率画框会**跳动**，不跟手 |
| 前端 `Dashboard.vue` | 收到结果只弹**顶部横幅** | 视频上没有任何框 |
| 前端 overlay | `overlayCanvas` 已覆盖在视频上（现画防区） | ✅ 可复用同一层画人脸框 |

---

## 1. 核心思路：后端给"关键位置"，前端插值出"丝滑"

**不提高后端检测频率**（否则加重卡顿）。改为：

```
后端：每次检测（~2次/秒）→ 推送当前所有人脸的【归一化坐标 + track_id + 姓名】
前端：requestAnimationFrame（~60fps）→ 每帧把框位置向"目标位置"插值(lerp)靠近
```

人脸不会瞬移，2 次/秒的关键位置 + 前端 60fps 插值补间，视觉上就是**平滑跟随**。
后端几乎零额外负担，前端只多一个 canvas 动画循环。

> 为什么不用后端跟踪器(KCF/CSRT)？那要在中间帧持续跑跟踪，反而加重后端。
> 前端插值把"平滑"这件事放到本就空闲的浏览器 GPU/rAF 上，是这个场景的最优解。
> （若后续人脸高速移动出现"追赶滞后"，再考虑进阶方案，见 §6。）

## 2. 后端改造

### 2.1 检测所有人脸（不止第一张）+ 分配稳定 track_id

`face.py` 现在 `_do_detect` 只处理 `face_rects[0]`。改为遍历所有 rect：
- 每张脸都算归一化坐标 `[x1/w, y1/h, x2/w, y2/h]`（与防区一致的归一化，前端好缩放）。
- 用**简单 IoU 跟踪**给每张脸分配稳定的 `track_id`，让"姓名"能稳定绑定到同一张脸上，
  不会因为帧间抖动而在两个人之间跳。
- **重活（活体 + 128维特征 + 会员匹配）仍只对主脸做**，保持算力不变；
  其余脸只给框（标记为"识别中"或沿用上次结果）。

新增一个轻量跟踪器（纯 Python，无模型，开销可忽略），放在 `face.py` 顶部：

```python
class _FaceTracker:
    """基于 IoU 的极简人脸跟踪，给每张脸分配稳定 track_id。"""
    def __init__(self, iou_thresh=0.3, max_missed=5):
        self._tracks = {}          # track_id -> {"box": [x1,y1,x2,y2], "missed": int, "name": str|None, "type": str}
        self._next_id = 1
        self._iou_thresh = iou_thresh
        self._max_missed = max_missed

    @staticmethod
    def _iou(a, b):
        ix1, iy1 = max(a[0], b[0]), max(a[1], b[1])
        ix2, iy2 = min(a[2], b[2]), min(a[3], b[3])
        iw, ih = max(0, ix2 - ix1), max(0, iy2 - iy1)
        inter = iw * ih
        area_a = (a[2] - a[0]) * (a[3] - a[1])
        area_b = (b[2] - b[0]) * (b[3] - b[1])
        union = area_a + area_b - inter
        return inter / union if union > 0 else 0.0

    def update(self, boxes):
        """boxes: 本帧检测到的像素框列表。返回 [(track_id, box), ...]。"""
        assigned, used = [], set()
        for box in boxes:
            best_id, best_iou = None, self._iou_thresh
            for tid, t in self._tracks.items():
                if tid in used:
                    continue
                iou = self._iou(box, t["box"])
                if iou >= best_iou:
                    best_id, best_iou = tid, iou
            if best_id is None:
                best_id = self._next_id
                self._next_id += 1
                self._tracks[best_id] = {"box": box, "missed": 0, "name": None, "type": None}
            else:
                self._tracks[best_id]["box"] = box
                self._tracks[best_id]["missed"] = 0
            used.add(best_id)
            assigned.append((best_id, box))
        # 老化：连续未命中的 track 删除
        for tid in list(self._tracks):
            if tid not in used:
                self._tracks[tid]["missed"] += 1
                if self._tracks[tid]["missed"] > self._max_missed:
                    del self._tracks[tid]
        return assigned

    def set_identity(self, track_id, type_, name):
        if track_id in self._tracks:
            self._tracks[track_id]["type"] = type_
            self._tracks[track_id]["name"] = name

    def get_identity(self, track_id):
        t = self._tracks.get(track_id, {})
        return t.get("type"), t.get("name")
```

在 `FaceDetector.__init__` 里加：`self._tracker = _FaceTracker()`。

### 2.2 在 `_do_detect` 里收集所有框并推送坐标

`_do_detect`（`:384`）改造要点（保持现有主脸识别逻辑不动，只在外围加"多脸框收集 + 推送"）：

```python
def _do_detect(self, frame):
    face_rects = self._matcher.detect_faces(frame.image)
    h, w = frame.image.shape[:2]

    # 1) 收集所有像素框 → 更新跟踪器
    pixel_boxes = [
        [max(0, r.left()), max(0, r.top()), min(w, r.right()), min(h, r.bottom())]
        for r in face_rects
    ]
    tracked = self._tracker.update(pixel_boxes)   # [(track_id, box)]

    # 2) 主脸（第一张）走原有的活体+特征+匹配逻辑 …（保持不变）
    #    识别出结果后，把身份绑定到主脸对应的 track_id：
    #    self._tracker.set_identity(main_track_id, result_type, name)

    # 3) 推送所有脸的框 + 身份（归一化坐标）
    faces_payload = []
    for tid, box in tracked:
        x1, y1, x2, y2 = box
        t_type, t_name = self._tracker.get_identity(tid)
        faces_payload.append({
            "track_id": tid,
            "box": [x1 / w, y1 / h, x2 / w, y2 / h],   # 归一化 [0,1]
            "type": t_type or "detecting",             # member / stranger / detecting
            "name": t_name,
        })
    self._push_faces(faces_payload)

    # …（下方原有 return AlarmEvent 逻辑不变）
```

> `main_track_id` = 主脸像素框在 `tracked` 里对应的 id（主脸就是 `pixel_boxes[0]`，
> 取 `tracked` 中 box 等于它的那个 tid）。

### 2.3 新增 `_push_faces` 广播（独立于现有横幅推送）

现有 `broadcast_face_result` 推的是"横幅结果"（顶部欢迎语），**语义不同，不要混用**。
新增一路专门推"框列表"，前端单独订阅：

`api/ws.py` 里新增（仿照现有 `face_result_queue`）：

```python
face_boxes_queue = queue.Queue(maxsize=2)   # 只保留最新，满了丢旧

def broadcast_face_boxes(faces: list) -> None:
    try:
        if face_boxes_queue.full():
            face_boxes_queue.get_nowait()      # 丢最旧，只推最新框
        face_boxes_queue.put_nowait({"type": "faces", "faces": faces})
    except Exception:
        pass
```

并在 `register_ws_routes` 加路由 `@sock.route("/ws/face_boxes")`，循环
`face_boxes_queue.get(timeout=0.5)` → `ws.send(json)`（与现有 `/ws/face_recognition` 同款写法）。

`face.py` 里 `_push_faces` 调用它：

```python
def _push_faces(self, faces):
    try:
        from ..api.ws import broadcast_face_boxes
        broadcast_face_boxes(faces)
    except Exception:
        pass
```

> 也可以复用现有 `/ws/face_recognition` 通道、加个 `type: "faces"` 分支，少开一条连接。
> 但独立通道更清晰、互不干扰横幅逻辑。二选一，推荐独立通道。

## 3. 前端改造（Dashboard.vue）— 平滑跟随核心

### 3.1 订阅框数据

在 `onMounted` 里连 `/ws/face_boxes`，把收到的 `faces` 存成"目标框"：

```js
let wsFaceBoxes = null
let targetFaces = []       // 后端最新推来的目标框（归一化坐标）
let renderFaces = []       // 当前正在渲染的框（每帧向 target 插值）

function connectFaceBoxesWs() {
  wsFaceBoxes = new WebSocket(`ws://${location.host}/ws/face_boxes`)
  wsFaceBoxes.onmessage = (e) => {
    const data = JSON.parse(e.data)
    if (data.type === 'faces') {
      targetFaces = data.faces || []
      syncRenderFaces()          // 按 track_id 对齐新增/移除
    }
  }
  wsFaceBoxes.onclose = () => { setTimeout(connectFaceBoxesWs, 3000) }
}
```

### 3.2 按 track_id 对齐（新增淡入、消失移除）

```js
function syncRenderFaces() {
  const map = new Map(renderFaces.map(f => [f.track_id, f]))
  const next = []
  for (const t of targetFaces) {
    const cur = map.get(t.track_id)
    if (cur) {
      cur.target = t.box                 // 更新目标位置
      cur.type = t.type; cur.name = t.name
      next.push(cur)
    } else {
      // 新脸：初始位置=目标位置（直接出现），可加 alpha 淡入
      next.push({ track_id: t.track_id, box: [...t.box], target: [...t.box],
                  type: t.type, name: t.name })
    }
  }
  renderFaces = next                     // 未在 target 中的 track 自然移除
}
```

### 3.3 rAF 插值循环（丝滑的关键）

```js
const LERP = 0.25   // 每帧向目标靠近 25%，越大越跟手、越小越顺滑
let rafId = null

function animateFaces() {
  const canvas = overlayCanvas.value
  if (canvas) {
    for (const f of renderFaces) {
      for (let i = 0; i < 4; i++) {
        f.box[i] += (f.target[i] - f.box[i]) * LERP    // 线性插值补间
      }
    }
    drawOverlay()        // 现有防区 + 新增人脸框一起画
  }
  rafId = requestAnimationFrame(animateFaces)
}
```

`onMounted` 里 `animateFaces()` 启动，`onUnmounted` 里 `cancelAnimationFrame(rafId)` +
`wsFaceBoxes?.close()`。

> **与前面的卡顿优化配合**：rAF 只在有人脸时才有绘制成本；无脸时 `renderFaces` 为空，
> `drawOverlay` 基本空转。若担心常驻 rAF，可在 `renderFaces.length === 0 && 无红区` 时
> 暂停循环、有数据时再启，和 flashTimer 同样的按需策略。

### 3.4 在 drawOverlay 里画框 + 名字

在现有 `drawOverlay()`（`:170`，画完防区后）追加：

```js
  // ---- 人脸框 + 身份标签 ----
  renderFaces.forEach((f) => {
    const [nx1, ny1, nx2, ny2] = f.box
    const x = nx1 * canvas.width, y = ny1 * canvas.height
    const bw = (nx2 - nx1) * canvas.width, bh = (ny2 - ny1) * canvas.height

    const isMember = f.type === 'member'
    ctx.save()
    ctx.strokeStyle = isMember ? '#67c23a' : (f.type === 'stranger' ? '#f56c6c' : '#e6a23c')
    ctx.lineWidth = 2
    ctx.strokeRect(x, y, bw, bh)

    // 标签：会员显示姓名，陌生人显示"陌生人"，未定显示"识别中"
    const label = isMember ? (f.name || '会员')
                : f.type === 'stranger' ? '陌生人' : '识别中…'
    ctx.font = '600 13px system-ui, sans-serif'
    const padX = 6, boxH = 20
    const textW = ctx.measureText(label).width
    ctx.fillStyle = ctx.strokeStyle
    ctx.fillRect(x, Math.max(0, y - boxH), textW + padX * 2, boxH)
    ctx.fillStyle = '#fff'
    ctx.textBaseline = 'middle'
    ctx.fillText(label, x + padX, Math.max(0, y - boxH) + boxH / 2)
    ctx.restore()
  })
```

坐标乘以 `canvas.width/height` 即可——因为后端归一化过，无论视频分辨率、
canvas 显示尺寸怎么变，框都自动贴合（和防区完全一样的处理）。

## 5. 验证方法

1. **单人**：走到镜头前，框贴住脸并随移动**平滑滑动**（非跳跃）；会员显示姓名，陌生人显示"陌生人"。
2. **多人**：两人同时入镜，各自一个框，姓名不串（靠 track_id 稳定绑定）。
3. **离开**：人走出画面，框在 ~5 次未命中后消失（`max_missed`）。
4. **性能**：Chrome Performance 录制，rAF 循环的 Scripting 占用应很小；后端 CPU 与改造前基本持平（未提高检测频率）。
5. **跟手 vs 顺滑**：调 `LERP`（0.15 更顺、0.35 更跟手）取平衡。

## 6. 进阶（非必需，视效果再定）

- **中间帧也更新框**：若 2次/秒 的关键位置在快速移动下仍觉滞后，可让后端在**不做识别的帧**
  只跑轻量 `detect_faces()`（比完整识别便宜）单独推框，把框刷新率提到 ~5-8/次秒，插值更贴。
  代价是后端负担上升，与卡顿目标冲突，**先不做**。
- **陌生人也做识别**：当前只主脸做重活。若要每张脸都判身份，需扩推理，评估算力后再定。
- **框内小抖动**：dlib bbox 逐帧有轻微抖动，插值已能平滑大部分；必要时对 target 框本身做一次 EMA。
