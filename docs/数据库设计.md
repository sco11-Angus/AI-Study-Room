# 数据库设计说明书

> 本文件是系统设计说明书 §8「数据模型设计」的落地细化，作为 D（数据层）建表与 B/C/E 落库的唯一依据。
> 关联：`backend/app/models/entities.py`（SQLAlchemy 模型）、`backend/app/models/db.py`（初始化，待建）。
> 数据库：开发用 SQLite，生产可切 MySQL 8（字段类型已给出两套映射）。

---

## 一、设计原则

- **主键**：统一自增整型 `id`（`member` 用 `member_id`，`camera`/`region` 语义主键亦为 id）。
- **外键**：跨表引用一律建外键约束；SQLite 需 `PRAGMA foreign_keys=ON`。
- **枚举**：用字符串枚举（SQLite 无原生 enum，用 `CHECK` 约束或应用层校验；MySQL 用 `ENUM`）。
- **时间**：统一 UTC，`datetime` 类型，`created_at`/`updated_at` 由应用或 DB 默认填充。
- **JSON**：多边形顶点、人脸特征等以 JSON 文本存储（SQLite `TEXT` / MySQL `JSON`）。
- **命名**：表名、字段名 `snake_case`，单数表名。

## 二、ER 关系图

```
                    ┌──────────┐
                    │ app_user │
                    └────┬─────┘
                         │ 1
                         │ N
┌──────────┐ 1     N ┌───▼──────┐ 1     N ┌────────────┐ N     1 ┌────────┐
│  camera  ├────────▶│  region  ├────────▶│ seat_status│         │ guard  │
└──────────┘         └────┬─────┘         └────────────┘         └───┬────┘
                          │ 1                                        │ 1
                          │ N                                        │ N
                     ┌────▼────────┐ 1      N ┌──────────────────┐   │
                     │ alarm_event ├─────────▶│ notification_log │◀──┘
                     └─────────────┘          └──────────────────┘

┌──────────┐ 1    1 ┌──────────────┐
│  member  ├───────▶│ (face 特征内嵌)│   member.feature 存 128 维向量
└──────────┘        └──────────────┘
```

关系说明：
- 一个 `camera` 有多个 `region`（防区/座位）。
- 一个 `region`（座位）有多条 `seat_status` 历史与多条 `alarm_event`。
- 一个 `alarm_event` 触发多条 `notification_log`（首发 + 升级）。
- 一个 `guard`（安全员）接收多条 `notification_log`。
- `member` 存注册会员及其人脸特征，供告警抓拍比对。

## 三、表结构定义

类型列格式：`SQLite 类型 / MySQL 类型`。

### 3.1 camera（摄像头）

| 字段 | 类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| id | INTEGER / INT | PK, AUTOINCREMENT | 摄像头 ID |
| name | TEXT / VARCHAR(128) | NOT NULL | 名称/安装位置 |
| stream_url | TEXT / VARCHAR(256) | NOT NULL | RTMP 拉流地址 |
| resolution | TEXT / VARCHAR(32) | — | 原始分辨率 `1920x1080`，用于坐标映射 |
| status | TEXT / ENUM | DEFAULT 'offline' | online / offline |
| created_at | DATETIME | DEFAULT now | 创建时间 |

索引：无（数据量小）。

### 3.2 region（防区 / 座位）

| 字段 | 类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| id | INTEGER / INT | PK, AUTOINCREMENT | 防区 ID |
| camera_id | INTEGER / INT | FK → camera.id, NOT NULL | 所属摄像头 |
| name | TEXT / VARCHAR(128) | NOT NULL | 防区/座位名 |
| type | TEXT / ENUM | NOT NULL | danger_zone / seat |
| polygon | TEXT / JSON | NOT NULL | 顶点数组（原始分辨率像素），如 `[[120,80],[400,80],...]` |
| x_distance | INTEGER / INT | DEFAULT 50 | 安全距离阈值（像素） |
| y_stay_time | INTEGER / INT | DEFAULT 10 | 允许危险停留时间（秒） |
| created_at | DATETIME | DEFAULT now | — |

索引：`idx_region_camera(camera_id)`。

### 3.3 app_user（自习用户）

> 原 §8 中 `seat_status.user_id` 引用的用户表，此前未定义，此处补全。

| 字段 | 类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| id | INTEGER / INT | PK, AUTOINCREMENT | 用户 ID |
| nickname | TEXT / VARCHAR(64) | — | 昵称 |
| device_token | TEXT / VARCHAR(256) | — | 弱提醒推送目标（专属屏幕/手机） |
| created_at | DATETIME | DEFAULT now | — |

### 3.4 seat_status（自习状态）

| 字段 | 类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| id | INTEGER / INT | PK, AUTOINCREMENT | — |
| user_id | INTEGER / INT | FK → app_user.id | 用户 |
| region_id | INTEGER / INT | FK → region.id | 座位 |
| status | TEXT / ENUM | NOT NULL | idle / studying / resting |
| updated_at | DATETIME | DEFAULT now, ON UPDATE now | 状态变更时间 |

索引：`idx_seat_region(region_id)`。
说明：`studying` 激活该座位疲劳检测，`resting` 挂起（§4.2）。

### 3.5 alarm_event（告警事件）

| 字段 | 类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| id | INTEGER / INT | PK, AUTOINCREMENT | 告警 ID |
| region_id | INTEGER / INT | FK → region.id | 触发防区/座位 |
| camera_id | INTEGER / INT | FK → camera.id | 触发摄像头（冗余，便于查询） |
| type | TEXT / ENUM | NOT NULL | intrusion / fire_smoke / occupy / fatigue / fight |
| snapshot_url | TEXT / VARCHAR(256) | — | 抓拍图路径 |
| face_match | TEXT / VARCHAR(64) | — | 会员匹配结果：`member:<id>` / `stranger` |
| level | INTEGER / INT | DEFAULT 1 | 0=弱提醒(私有) 1=普通 2+=升级/高优先(如打架) |
| status | TEXT / ENUM | DEFAULT 'pending' | pending / notified / confirmed / escalated |
| extra | TEXT / JSON | — | 附加信息（如打架的 vis_score/aud_score/fuse、涉事人员框） |
| created_at | DATETIME | DEFAULT now | 触发时间 |
| confirmed_at | DATETIME | — | 确认时间 |

索引：`idx_alarm_status(status)`、`idx_alarm_created(created_at)`、`idx_alarm_region_type(region_id, type)`（去重查询用）。

### 3.6 member（会员 + 人脸特征）

| 字段 | 类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| member_id | INTEGER / INT | PK, AUTOINCREMENT | 会员 ID |
| name | TEXT / VARCHAR(128) | NOT NULL | 姓名 |
| feature | TEXT / JSON | — | 128 维人脸特征向量（JSON 数组） |
| created_at | DATETIME | DEFAULT now | — |

说明：告警抓拍时裁剪面部提特征，与本表逐条欧氏距离比对（阈值 0.6）。

### 3.7 guard（安全员）

> 原 §8 ER 图中出现但未定义，此处补全。

| 字段 | 类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| id | INTEGER / INT | PK, AUTOINCREMENT | 安全员 ID |
| name | TEXT / VARCHAR(128) | NOT NULL | 姓名 |
| dingtalk_id | TEXT / VARCHAR(128) | — | 钉钉用户标识（@ 用） |
| role | TEXT / ENUM | DEFAULT 'primary' | primary(主责) / leader(科长/负责人) |
| priority | INTEGER / INT | DEFAULT 0 | 升级顺序，越小越先通知 |

说明：超时升级时按 `role`/`priority` 找下一级接收人（§7.4）。

### 3.8 notification_log（钉钉通知记录）

| 字段 | 类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| id | INTEGER / INT | PK, AUTOINCREMENT | — |
| alarm_id | INTEGER / INT | FK → alarm_event.id | 关联告警 |
| guard_id | INTEGER / INT | FK → guard.id | 接收安全员 |
| stage | TEXT / ENUM | NOT NULL | primary(首发) / escalated(升级) |
| sent_at | DATETIME | DEFAULT now | 发送时间 |
| ack_at | DATETIME | — | 确认时间 |

索引：`idx_notify_alarm(alarm_id)`。

## 四、枚举字典

| 枚举 | 取值 | 含义 |
| --- | --- | --- |
| camera.status | online / offline | 在线 / 离线 |
| region.type | danger_zone / seat | 危险防区 / 独立座位 |
| seat_status.status | idle / studying / resting | 空闲 / 自习中 / 休息中 |
| alarm_event.type | intrusion / fire_smoke / occupy / fatigue / fight | 闯入 / 烟火 / 占座 / 疲劳 / 打架(音视频融合) |
| alarm_event.status | pending / notified / confirmed / escalated | 待处理 / 已通知 / 已确认 / 已升级 |
| alarm_event.level | 0 / 1 / 2+ | 弱提醒(私有) / 普通 / 升级后 |
| guard.role | primary / leader | 主责安全员 / 科长负责人 |
| notification_log.stage | primary / escalated | 首发通知 / 升级通知 |

## 五、建表 DDL（SQLite 参考）

```sql
PRAGMA foreign_keys = ON;

CREATE TABLE camera (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    name        TEXT NOT NULL,
    stream_url  TEXT NOT NULL,
    resolution  TEXT,
    status      TEXT DEFAULT 'offline' CHECK(status IN ('online','offline')),
    created_at  DATETIME DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE region (
    id           INTEGER PRIMARY KEY AUTOINCREMENT,
    camera_id    INTEGER NOT NULL REFERENCES camera(id),
    name         TEXT NOT NULL,
    type         TEXT NOT NULL CHECK(type IN ('danger_zone','seat')),
    polygon      TEXT NOT NULL,
    x_distance   INTEGER DEFAULT 50,
    y_stay_time  INTEGER DEFAULT 10,
    created_at   DATETIME DEFAULT CURRENT_TIMESTAMP
);
CREATE INDEX idx_region_camera ON region(camera_id);

CREATE TABLE app_user (
    id            INTEGER PRIMARY KEY AUTOINCREMENT,
    nickname      TEXT,
    device_token  TEXT,
    created_at    DATETIME DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE seat_status (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id     INTEGER REFERENCES app_user(id),
    region_id   INTEGER REFERENCES region(id),
    status      TEXT NOT NULL CHECK(status IN ('idle','studying','resting')),
    updated_at  DATETIME DEFAULT CURRENT_TIMESTAMP
);
CREATE INDEX idx_seat_region ON seat_status(region_id);

CREATE TABLE alarm_event (
    id            INTEGER PRIMARY KEY AUTOINCREMENT,
    region_id     INTEGER REFERENCES region(id),
    camera_id     INTEGER REFERENCES camera(id),
    type          TEXT NOT NULL CHECK(type IN ('intrusion','fire_smoke','occupy','fatigue','fight')),
    snapshot_url  TEXT,
    face_match    TEXT,
    level         INTEGER DEFAULT 1,
    status        TEXT DEFAULT 'pending' CHECK(status IN ('pending','notified','confirmed','escalated')),
    extra         TEXT,
    created_at    DATETIME DEFAULT CURRENT_TIMESTAMP,
    confirmed_at  DATETIME
);
CREATE INDEX idx_alarm_status ON alarm_event(status);
CREATE INDEX idx_alarm_created ON alarm_event(created_at);
CREATE INDEX idx_alarm_region_type ON alarm_event(region_id, type);

CREATE TABLE member (
    member_id   INTEGER PRIMARY KEY AUTOINCREMENT,
    name        TEXT NOT NULL,
    feature     TEXT,
    created_at  DATETIME DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE guard (
    id           INTEGER PRIMARY KEY AUTOINCREMENT,
    name         TEXT NOT NULL,
    dingtalk_id  TEXT,
    role         TEXT DEFAULT 'primary' CHECK(role IN ('primary','leader')),
    priority     INTEGER DEFAULT 0
);

CREATE TABLE notification_log (
    id        INTEGER PRIMARY KEY AUTOINCREMENT,
    alarm_id  INTEGER REFERENCES alarm_event(id),
    guard_id  INTEGER REFERENCES guard(id),
    stage     TEXT NOT NULL CHECK(stage IN ('primary','escalated')),
    sent_at   DATETIME DEFAULT CURRENT_TIMESTAMP,
    ack_at    DATETIME
);
CREATE INDEX idx_notify_alarm ON notification_log(alarm_id);
```

## 六、初始化与种子数据（D 负责）

- `backend/app/models/db.py`：`create_engine` + `SessionLocal` + `init_db()`（`Base.metadata.create_all`）。
- `backend/app/models/seed.py`：插入 1 个测试 camera、若干 region、1 个 app_user、1 个 primary guard，便于 B/C/E/F 联调。
- 首次启动在 `create_app()` 内调用 `init_db()`（若表不存在则建）。

## 七、与设计说明书的差异记录

| 项 | 说明 |
| --- | --- |
| 新增 `app_user` 表 | §8 ER 图引用了 `User` 但未给字段，此处补全 |
| 新增 `guard` 表 | §8 ER 图引用了 `Guard` 但未给字段，此处补全 |
| `alarm_event` 增 `camera_id` | 冗余字段，便于按摄像头查询与抓拍定位 |
| `alarm_event.type` 增 `fight` | 音视频融合打架检测（D 模块） |
| `alarm_event` 增 `extra` (JSON) | 存打架的双模置信度/涉事人员框等附加信息 |
| 补充索引 | 针对告警查询、去重、外键关联加索引 |

> 若后续需求确认「只做人员检测、不做人脸重识别」，则 `member` 表与 `alarm_event.face_match` 字段可整体移除——目前保留，与现行 `entities.py` 及设计说明书 §7.2 一致。
