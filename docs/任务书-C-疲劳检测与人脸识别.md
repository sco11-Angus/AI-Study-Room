# C 任务书 — 疲劳检测 + 自习伴侣 + 人脸识别

> 角色定位：疲劳与人脸均基于 Dlib，技术栈一致，合并给一人。
> 关联设计：系统设计说明书 §4、§7.2。

## 一、职责范围

1. **疲劳检测（模块一，§4.3）**
   - Dlib 68 点人脸关键点定位。
   - 复用 `detectors/fatigue.py` 的 EAR/MAR 公式：EAR < 阈值持续 ≥ 2s（闭眼）或 MAR 超标（打哈欠）。
   - 包装成 A 的 `Detector` 插件注册，**不自建线程池**。
   - 触发**弱提醒**（推该用户专属屏幕/手机，不拉公用警报）。
2. **自习状态宣告（模块一，§4.2）**
   - `api/seat_status`：`studying` → 激活该座位疲劳检测；`resting` → 立即挂起并释放算力。
   - 通过 A 的引擎按座位启停对应检测器。
3. **人脸识别（模块四抓拍部分，§7.2）**
   - 完善 `detectors/face.py`：128 维特征提取 + 会员库欧氏距离匹配，输出「会员ID / stranger」。
   - 供 E 在告警抓拍时调用。

## 二、交付物

| 交付 | 说明 |
| --- | --- |
| 疲劳检测插件 | EAR/MAR 判定，产出弱提醒事件 |
| seat_status 接口 | 状态切换驱动算法激活/挂起 |
| 人脸匹配服务 | `match(feature) -> 会员ID/stranger`，供 E 调用 |

## 三、验收标准

- [ ] `studying` 状态下闭眼 2s 触发弱提醒，不触发公用警报。
- [ ] `resting` 状态下趴桌/闭眼均不干扰，且该区域算力被释放。
- [ ] 人脸匹配能区分注册会员与陌生人。

## 四、协作接口

- **上游**：A（帧 + Detector 接口）。
- **下游**：E（人脸匹配结果、疲劳事件）、D（状态落库）、F（伴侣页面 + 弱提醒展示）。
