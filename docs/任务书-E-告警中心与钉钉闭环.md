# E 任务书 — 告警中心 + 钉钉逐级闭环（模块四主体，加分项）

> 角色定位：告警枢纽。需第一周牵头定义「统一告警事件结构」，B/C/D 触发告警都往此结构塞，F 据此渲染。
> 关联设计：系统设计说明书 §7、§10.2。

## 一、职责范围

1. **统一告警事件结构（第一周冻结）**
   - 定义所有检测器共用的告警事件格式（type / region_id / snapshot / face_match / level / created_at 等，见 §8.2 alarm_event）。
2. **告警服务（§7.1、§7.2、§10.2）**
   - 完善 `services/alarm.py`：告警状态机、现场抓拍落盘、裁剪面部（调 C 的人脸匹配）、同防区同类型冷却去重。
3. **看板异动（§7.3）**
   - WebSocket 告警推送（`api/ws.py` + flask-sock），推给前端 F 触发格子红闪 + 蜂鸣。
4. **钉钉逐级上报（§7.4）**
   - 完善 `services/dingtalk.py`：群机器人 Webhook 发 ActionCard 卡片。
   - 主责安全员 3 分钟内未点击确认 → 自动升级推送科长/负责人。
   - `api/alarms/{id}/confirm` 确认回调 → 取消升级计时。

## 二、交付物

| 交付 | 说明 | 时间要求 |
| --- | --- | --- |
| 统一告警事件结构 | 全体检测器遵循 | **第一天冻结** |
| 告警服务 | 抓拍 + 去重 + 状态机 | 中期 |
| WebSocket 推送 | 前端实时收告警 | 联调阶段 |
| 钉钉闭环 | 卡片 + 3min 超时升级 + 确认 | 联调阶段 |

## 三、验收标准

- [ ] 告警瞬间抓拍并调人脸匹配，标注会员/陌生人。
- [ ] 同防区同类型冷却窗口内不重复刷屏。
- [ ] 前端格子实时红闪 + 蜂鸣。
- [ ] 主责 3 分钟未确认自动升级；点击确认后停止升级。

## 四、协作接口

- **上游**：B/C/D（告警事件）、C（人脸匹配）、A（抓拍帧）。
- **下游**：F（WebSocket 推送渲染）、钉钉群机器人。
