# D2 任务书 — 情感分析增强打架检测（D 模块增量）

> 角色定位：D 模块的精度升级。现状音频侧只识别「音量/噪度」，分不清欢呼与怒吼，导致庆祝、鼓掌、追逐嬉闹等欢乐场景误报打架。本任务引入**人脸表情 + 声学情绪**双通道，作为告警**闸门**，把「情绪」纳入语义判断，主攻**降低假阳性**。
> 关联设计：系统设计说明书 §3（音轨）、§7（告警闭环）；[[任务书-D-音视频打架检测]]。
> 涉及文件：`backend/app/detectors/emotion.py`(新建)、`backend/app/detectors/fight.py`(改)、`backend/app/config.py`(加参数)、`backend/run.py`(启用 FightPlugin)。
> 依赖：D 模块已落地的 `fight.py`/`audio.py`；B 的 dlib 人脸框（`FaceBoxProvider`，复用不重复检测）；音轨管线已打通（`scheduler._audio_loop` → `feed_audio`，无需改动）。
> 选型已定：表情用 **HSEmotion(ONNX, CPU)**；声学用 **YAMNet(tflite, CPU)**。torch 虽在环境中，音频侧仍先用 YAMNet（管道已就绪、改动最小），`AudioConflict` 抽象为可换后端，预留 wav2vec2 升级。

## 一、模块目标

在现有音视频融合的基础上，增加「情绪一致性」判据：只有当画面存在**贴身+高速运动**、音频存在**尖叫/怒吼**、**且情绪呈现愤怒/恐惧**时才确认打架。情绪不作第三个加分项，而作**调制闸门**，直接压制欢乐场景的视觉冲突分。

```
        欢呼庆祝 + 追逐嬉闹 + 大家在笑   → emo_gate≈0.1 → vis 打 0.4 折 → 不报
        肢体冲突 + 怒吼尖叫 + 愤怒/恐惧   → emo_gate≈0.9 → vis 保留放大 → 报警

    vis' = vis × (0.4 + 0.6 × emo_gate)          emo_gate 由 愤怒/恐惧/尖叫 驱动
    aud' = YAMNet(scream/shout/yell) 概率主导     取代原「有多响」
    fuse = 0.6*vis' + 0.4*aud'                    公式结构不变，只换输入含义
```

## 三、任务拆解

### 任务 D2-0：启用现有打架链路（前提，勿跳过）
- `run.py:33` 取消注释注册 `FightPlugin`，`person_provider` 用 `SharedContextProvider(engine.shared_ctx)`（无 B 人员框时退回 `FaceBoxProvider`）。
- 跑 `tests/test_fight_integration.py` 确认音视频链路活着，再往上加情绪。

### 任务 D2-A：视觉情绪通道（新建 `detectors/emotion.py`）
- `FacialEmotion` 单例：`setup()` 加载 HSEmotion ONNX（`onnxruntime`，provider 由 `EMOTION_DEVICE` 决定 CPU/GPU）。
- `score(image, boxes) -> float`：对每个人脸框裁图 → 推理 → 取 `anger+fear` 概率，返回全场峰值 ∈ [0,1]。
- 无框 / 模型缺失 / 异常 → 返回 `1.0`（中性放行，等价当前行为），绝不抛异常拖垮引擎。

### 任务 D2-B：音频情绪通道（改 `fight.py::AudioConflict`，约 fight.py:147）
- `AudioConflict` 抽象出可换后端；默认 `YamnetBackend`：吃 `AudioChunk.pcm`(16kHz)，输出 521 类，取打斗相关类求和。
  - 命中类：`Screaming/Shout/Yell/Children shouting/Crying, sobbing/Glass/Breaking`。
- 融合：`aud = 0.7*aud_emo + 0.3*loud`（情绪主导，响度托底，防 YAMNet 抖动）。
- **调度器不动**：`feed_audio` 那根线已经在流数据。

### 任务 D2-C：融合闸门（改 `fight.py::FusionDebouncer.update`，fight.py:184）
- 签名加 `emo_gate`：`vis_g = vis_score * (0.4 + 0.6*emo_gate)`；`fuse = w_v*vis_g + w_a*aud_score`。
- 双模 AND 与持续性防抖逻辑保持不变。

### 任务 D2-A'：FightPlugin 接线（改 fight.py:219/228/239）
- `__init__` 建 `FacialEmotion`；`setup()` 内 load；`detect()` 内 `emo_gate = self._emotion.score(frame.image, boxes)` 传入 `fusion.update(...)`。
- 告警 `extra` 增加 `emo_gate` 与命中情绪标签，供 F 大屏 / G 取证展示「因愤怒+尖叫确认」。

### 任务 D2-D：参数入 Config + 依赖
- `config.py` 加：`EMOTION_ENABLE`(默认 false 灰度)、`EMOTION_MODEL_PATH`、`YAMNET_MODEL_PATH`、`EMOTION_DEVICE`(cpu/gpu)、闸门系数、音频情绪权重。
- `requirements.txt` 加：`onnxruntime`、`tflite-runtime`(或退回 onnx 版 YAMNet)。权重放 `backend/model_weights/`，不进 git。
