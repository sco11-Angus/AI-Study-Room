# Alarm Close Loop

## Purpose
将检测到的异常事件通过抓拍、人脸匹配、持久化、看板展示、钉钉通知与人工确认形成可追溯的完整告警闭环，并保证事件状态能够在前后端实时同步。

## Requirements

### Requirement: 告警闭环处置
The system SHALL 在检测到异常事件时，执行抓拍、人脸匹配、看板展示和钉钉通知的完整闭环。

#### Scenario: 告警触发闭环
- **GIVEN** 检测器触发了一个告警事件
- **WHEN** 告警进入处置流程
- **THEN** 系统执行抓拍、人脸匹配、看板更新和钉钉通知

#### Scenario: 钉钉逐级上报
- **GIVEN** 一个已触发的告警
- **WHEN** 告警级别达到上报阈值
- **THEN** 钉钉机器人发送告警通知，包含抓拍图片和事件信息

#### Scenario: 告警回放
- **GIVEN** 一个已处置的告警事件
- **WHEN** 用户在看板上点击回放
- **THEN** 系统播放该告警时间段的视频片段
