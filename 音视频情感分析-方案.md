# 音视频打架检测 · 情感分析增强 — 实现方案

> 目标：把 D 模块音频侧从"只识别音量"升级为"识别人的情绪"，
> 用**表情 + 声学情绪**双通道当作打架告警的**闸门**，
> 主要解决**欢呼/嬉闹/鼓掌等假阳性导致的打架误报**。
>
> 首选模型组合（CPU 实时、零 torch 重依赖）：**HSEmotion（人脸表情）+ YAMNet（声学事件/尖叫）**。

---

## 0. 现状（为什么现在会误报）

当前 `backend/app/detectors/fight.py` 的音频侧 `AudioConflict.score()` 只算三种**纯声学**特征：

```
RMS 能量(dBFS)  +  过零率(ZCR)  +  谱质心(centroid)
     ↓                ↓                ↓
   "有多响"        "有多噪"        "频率有多高"
```

问题：**分不清情绪**。庆祝欢呼、鼓掌、放音乐、孩子追逐嬉闹都会把 `aud_score` 拉满；
只要视觉侧同时检测到"贴身 + 快速移动"（打闹、追逐同样满足），融合分就过阈值 → 误报。

```
现状融合公式（fight.py:185）
    fuse = 0.6*vis + 0.4*aud
           ↑ 贴身快速移动    ↑ 只是"很吵"
    → 欢乐场景与打架场景在特征上几乎无法区分
```

**架构已经把"情绪原料"准备好了**，接入成本极低：

```
┌──────────────────────────────────────────────────────────────┐
│  音轨管线(audio.py) → 16kHz 单声道 PCM / 1s 窗口                │
│                       ↑ 正是 YAMNet 的标准输入，直接喂          │
│                                                                │
│  FaceBoxProvider     → dlib 已框好每张人脸(face rects)         │
│                       ↑ 直接裁脸喂 FER 表情模型，无需重复检测    │
└──────────────────────────────────────────────────────────────┘
```

---

## 1. 核心思路：情绪当"闸门"，不当"第三个加分项"

直觉做法是 `fuse = w_v*vis + w_a*aud + w_e*emo`，把情绪当第三项加权。
**不推荐**——那样会稀释信号，且情绪分本身噪声大时反而拉高误报。

更稳的做法：让情绪作为**调制闸门 (gate)**，直接压制/放大已有的视觉与音频信号。

```
        场景A: 吵 + 快速移动 + 大家在笑      → emo_gate≈0.1 → 压制，不报
        场景B: 吵 + 快速移动 + 愤怒/恐惧/尖叫  → emo_gate≈0.9 → 放大，报警

    emo_gate = 由「愤怒 anger / 恐惧 fear / 尖叫 scream」三种负面强情绪驱动 ∈ [0,1]

    vis' = vis × (0.4 + 0.6 × emo_gate)      # 有负面情绪才认可视觉冲突
    aud' = 声学情绪概率(scream/shout/yell)    # 取代原来的"有多响"
    fuse = 0.6*vis' + 0.4*aud'                # 公式结构不变，只换输入含义
```

要点：`emo_gate` 低时把 `vis` 打到 0.4 倍（欢乐场景的"贴身快速移动"被大幅压制），
高时保留甚至放大。这样**直接干掉最大误报源，而不是叠加一个新信号**。

## 2. 推荐的已训练模型

两条模态各选"轻量能落地"的首选，各留一个"更强但更重"的备选。

| 模态 | 模型 | 输出 | 大小/运行 | 选它的理由 |
|------|------|------|-----------|-----------|
| **人脸表情** ⭐ | **HSEmotion**（EfficientNet-B0，ONNX） | 8类：anger/fear/disgust/happy/sad/surprise/neutral/contempt | ~15MB，ONNXRuntime CPU，<10ms/脸 | AffectNet SOTA 级，pip 直装，直接吃 dlib 人脸框 |
| 表情备选 | **FER+**（emotion-ferplus，ONNX Model Zoo） | 8类情绪 | ~35MB，纯 ONNX | 微软经典，零框架依赖，只需 onnxruntime |
| **声学事件/尖叫** ⭐ | **YAMNet**（TF-Hub / tflite） | AudioSet 521类，含 Screaming/Shout/Yell/Crying/Glass break | ~4MB，tflite CPU | 直接吃 16kHz PCM，事件概率即"打斗声学证据" |
| 声学备选（更细） | **wav2vec2 SER**（`audeering/wav2vec2-large-robust-...-msp-dim`） | 维度情绪：唤醒度 arousal / 效价 valence / 支配度 | ~300MB，需 torch | "唤醒度高 + 效价负" = 愤怒/恐惧，比分类更鲁棒 |

⭐ **首选组合：HSEmotion + YAMNet**。都轻、都能 CPU 实时、都精准命中已有数据流入口，
不引入 torch 重依赖（保持 `requirements.txt` 精简）。

**关于 GPU（部署环境未定）**：本方案按纯 CPU 设计即可上线；
若日后确认有 GPU，音频侧可平滑切换到 wav2vec2 维度情绪（更鲁棒），
表情侧 HSEmotion 也有 GPU provider，改一行 ONNXRuntime provider 即可，下游不动。

```
              情绪信号来源与角色
    ┌─────────────────────────────────────────────┐
    │  HSEmotion  ── 每张脸 ──▶ 愤怒/恐惧 概率      │──┐
    │  (吃 dlib 框)                                │  │  取 max/聚合
    │                                              │  ├─▶ emo_gate ─▶ 调制 vis
    │  YAMNet     ── 1s窗 ──▶ scream/shout/yell   │──┘             └─▶ 取代 aud
    │  (吃 16k PCM)                                │
    └─────────────────────────────────────────────┘
```

## 3. 三个精确接入点（改在哪）

全部落在 `backend/app/detectors/` 内，遵循任务书 D 的红线：
**复用 B 的人脸框、不重复加载模型、不自建线程、注册进 A 的引擎**。

```
① 视觉情绪  新增 emotion.py::FacialEmotion   —— 读 FaceBoxProvider 的框裁脸，出愤怒/恐惧概率
② 音频情绪  改 fight.py::AudioConflict.score —— 用 YAMNet 事件概率取代/增强"响度"
③ 融合闸门  改 fight.py::FusionDebouncer     —— emo_gate 调制 vis，而非叠加
```

### 3.1 视觉情绪（新增 `detectors/emotion.py`）

关键：**不新做人脸检测**。`FaceBoxProvider`（`person_source.py:60`）已经用 dlib 框好了每张脸，
`FightPlugin.detect()` 里已有 `boxes`。表情模型只负责"裁脸 → 出情绪概率"。

```python
# detectors/emotion.py（示意，非落地代码）
class FacialEmotion:
    """吃人脸框，返回该帧「负面强情绪」概率 ∈ [0,1]。
    单例加载 HSEmotion ONNX；CPU provider；不自建线程。"""
    NEG = ("anger", "fear")          # 驱动打架闸门的负面情绪
    def score(self, image, boxes) -> float:
        if not boxes:
            return 0.0
        peak = 0.0
        for (x1, y1, x2, y2) in boxes:
            crop = image[int(y1):int(y2), int(x1):int(x2)]
            probs = self._model.predict(crop)      # dict{label: p}
            peak = max(peak, sum(probs[k] for k in self.NEG))
        return peak                                 # 全场最"负面"的那张脸
```

### 3.2 音频情绪（改 `fight.py::AudioConflict.score`，约 fight.py:147）

把"有多响"升级为"是不是尖叫/怒吼"。YAMNet 吃 16kHz PCM（正是 `AudioChunk.pcm`），
输出 521 类概率，取其中打斗相关类求和：

```python
FIGHT_CLASSES = ("Screaming", "Shout", "Yell", "Children shouting",
                 "Crying, sobbing", "Glass", "Breaking")
aud_emo = sum(yamnet_probs[c] for c in FIGHT_CLASSES)
# 与原声学分融合：情绪主导，响度托底（避免 YAMNet 抖动时全靠它）
aud = 0.7 * aud_emo + 0.3 * loud
```

### 3.3 融合闸门（改 `fight.py::FusionDebouncer.update`，fight.py:184）

`FightPlugin` 每帧把 `emo_gate = FacialEmotion.score(frame.image, boxes)` 算好传进来：

```python
def update(self, vis_score, aud_score, emo_gate, ts):
    vis_g = vis_score * (0.4 + 0.6 * emo_gate)   # 无负面情绪 → 视觉打 0.4 折
    fuse  = self.w_v * vis_g + self.w_a * aud_score
    # 双模 AND 保持不变；欢乐场景 emo_gate≈0，vis_g 被压，难过阈值
    ...
```

告警 `extra` 里加上 `emo_gate` 与命中的情绪标签，前端看板/取证可展示"因愤怒+尖叫确认"。

## 5. 验证方法

- **单元**：`tests/test_emotion.py` — 喂欢笑脸图 → `emo_gate` 低；愤怒/恐惧脸图 → 高。
- **音频**：喂尖叫/鼓掌两段 wav → YAMNet 打斗类概率明显区分。
- **端到端（关键）**：用两段真实素材跑 `FightPlugin`：
  - ❌ 欢呼庆祝 + 追逐嬉闹 → **不告警**（这是本次核心目标）
  - ✅ 真实肢体冲突 + 怒吼/尖叫 → 告警，`extra.emo_gate` 高
- **性能**：确认 CPU 上表情 <10ms/脸、YAMNet <30ms/窗，不拖垮 A 的推理帧率。

## 6. 落地顺序

1. 放权重 + 加依赖 + config 开关（默认 `EMOTION_ENABLE=false`，可灰度）。
2. 写 `emotion.py::FacialEmotion`（HSEmotion），单测过。
3. `FightPlugin.detect` 接入 `emo_gate` 调制，跑端到端"欢乐不报"用例。
4. `AudioConflict` 接 YAMNet，替换响度主导。
5. 告警 `extra` 带情绪标签，前端看板展示确认依据。
6. （可选，确认有 GPU 后）音频侧切 wav2vec2 维度情绪，provider 改 GPU。

## 7. 开放问题

- **YAMNet 运行时**：`tflite-runtime` 在你的部署 Linux 上是否好装？装不动则退回 ONNX 版 YAMNet。
- **多脸聚合策略**：取全场最负面的一张（当前设计）还是打架框内人脸优先？后者更准但需人脸框与人员框关联。
- **闸门系数 `0.4 + 0.6`**：需用真实素材调；起点保守（宁可漏报欢乐误报，也别漏真打架）。
